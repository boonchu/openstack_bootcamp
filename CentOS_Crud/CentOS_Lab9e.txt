\f0\fs24 \cf0 Lab 9e: Create and Manage Volumes\
Volumes are block storage devices that you attach to instances to enable persistent storage. You can attach a volume to a running instance or detach a volume and attach it to another instance at any time. You can also create a snapshot from or delete a volume. Only administrative users can create volume types.\
Create a Volume\
1. Log in to the dashboard, choose a project, and click\'a0Volumes.\
2. Click\'a0Create Volume.\
   In the dialog box that opens, enter or select the following values.\
Volume Name\
Specify a name for the volume.\
Description\
Optionally, provide a brief description for the volume.\
Type\
Leave this field blank.\
Size (GB)\
The size of the volume in gigabytes.\
Volume Source\
Select one of the following options:\
No source, empty volume\
Creates an empty volume.\
An empty volume does not contain a file system or a partition table.\
Snapshot\
If you choose this option, a new field for\'a0Use snapshot as a source\'a0displays. You can select the snapshot from the list.\
Image\
If you choose this option, a new field for\'a0Use image as a sourcedisplays. You can select the image from the list.\
Select the\'a0Availability Zone\'a0from the list. By default, this value is set to the availability zone given by the cloud provider (for example, us-west\'a0or\'a0apac-south). For some cases, it could be\'a0nova.\
Volume\
If you choose this option, a new field for\'a0Use volume as a source displays. You can select the volume from the list.\
Options to use a snapshot or a volume as the source for a volume are displayed only if there are existing snapshots or volumes. \
\
   The dashboard shows the volume on the\'a0Volumes\'a0tab.\
Attach a Volume to an Instance\
After you create one or more volumes, you can attach them to instances. You can attach a volume to one instance at a time.\
1. Log in to the dashboard, choose a project, and click\'a0Volumes.\
2. Select the volume to add to an instance and click\'a0Edit Attachments.\
3. In the\'a0Manage Volume Attachments\'a0dialog box, select an instance.\
4. Enter the name of the device from which the volume is accessible by the instance.\
The actual device name might differ from the volume name because of hypervisor settings.\
5. Click\'a0Attach Volume.\
   The dashboard shows the instance to which the volume is now attached and the device name.\
You can view the status of a volume in the\'a0Volumes\'a0tab of the dashboard. The volume is either Available or In-Use.\
Now you can log in to the instance and mount, format, and use the disk.\
Detach a Volume from an Instance\
1. Log in to the dashboard, choose a project, and click\'a0Volumes.\
2. Select the volume and click\'a0Edit Attachments.\
3. Click\'a0Detach Volume\'a0and confirm your changes.\
A message indicates whether the action was successful.\
Create a Snapshot from a Volume\
1. Log in to the dashboard, choose a project, and click\'a0Volumes.\
2. Select a volume from which to create a snapshot.\
3. From the\'a0More\'a0lists, select\'a0Create Snapshot.\
4. In the dialog box that opens, enter a snapshot name and a brief description.\
5. Confirm your changes.\
      The dashboard shows the new volume snapshot in\'a0Volume Snapshots\'a0tab.\

Delete a Volume\
When you delete an instance, the data in its attached volumes is not destroyed.\
\'95 Log in to the dashboard, choose a project, and click\'a0Volumes.\
\'95 Select the check boxes for the volumes that you want to delete.\
\'95 Click\'a0Delete Volumes\'a0and confirm your choice.\
